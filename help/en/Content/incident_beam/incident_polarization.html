Incident polarization

Usually, ADDA performs simulation for two incident polarizations e0 (§11.1), which are then used to compute amplitude and Mueller scattering matrices (§11.2, §11.3). The latter can then be used to calculate the result for any incident polarization. Symmetry can sometimes be used to effectively avoid a second simulation (§6.7).

However, in certain applications one needs only the result for a single specific incident polarization, or the incident beam (e.g. point-dipole one) does not have a clear meaning of polarization at all. Currently, ADDA cannot handle such cases in a proper way,[44] but one can use “‑sym enf” to perform a single run for y‑polarization (§6.7). Then the scattering quantities that are produced for each incident polarization independently (they have Y in the filename) will be correct. But the scattering matrices albeit computed by ADDA are expected to be wrong, and can be turned off by "‑scat_matr none" (§11.3). Still, one can obtain the scattered intensity from the amplitude matrix, but the particular formulae depend on the scattering direction (§11.1, §11.3). For example, for default propagation direction and scattering in the yz‑plane S2 and S4 correspond to parallel and perpendicular components of the scattered fields (with respect to the scattering plane).
